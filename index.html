Définition

Une méthode de programmation qui préfère l'utilisation de fonctions aux changements d'état.
1e comparaison concrète for -> foreach 

var notes = [ 'do', 'ré', 'mi', 'fa' ];
for (var i = 0; i < 10; i++) {
  console.log(notes[i]);
}
vs 

notes.forEach(function(fruit){
  console.log(fruit);
})
Quelle différence avec la POO ? - on essaie d'éviter les changements d'état sur un objet donné, pour limiter les effets de bords

Functions as first class citizen

Dans un langage traitant les fonctions comme des "citoyens de 1ère classe", les fonctions peuvent être passées en paramètre (et en retour) d'autres fonctions. 

function octave () {
  return function (note) {
    return note + 8;
  }
}
octave(1) //  9
octave(2) // 10

Collections

filter()

Applique une fonction à chaque élément, et
retourne tous ceux pour lesquels cette fonction renvoie True. 

var partition = [ 42, 24, 91, 14, 27, 34 ];

partition.map(function (note){
  return note < 30;
})

// [ 24, 14, 27 ]

find()

Applique une fonction à chaque élément, et
retourne le premier pour lequel cette fonction renvoie True. 

var partition = [ 42, 24, 91, 14, 27, 34 ];

partition.map(function (note){
  return note < 30;
})

// 24

map()

Applique une fonction à chaque élément, et
retourne les résultats. 

var partition = [ 42, 24, 91, 14, 27, 34 ];

partition.map(octave)

// [ 50, 32, 99, 22, 35, 42 ]
