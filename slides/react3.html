  • Autre concept, Functional Reactive Programming ex: bacon.js
- stateless functional components: nerf de la guerre dans react = la gestion des states. Très sensible aux effets de bords. Facebook encourage l'écriture de petits composants appelés stateless functional, dont la sortie dépend totalement des paramètres en entrée
- état immuable, on réaffiche lorsque l'état change. Prob de savoir si l'état a changé, avec état immuable on le sait
- Undo: est-ce qu'on vous a déjà demandé de rajouter une fonctionnalité de type Undo dans une appli web après que le dev ait commencé ? La galère! Avec React et Immutable, c'est cadeau. Même le time travel!
- bacon.js: functional reactive programming, travaille sur des flux d'événements
