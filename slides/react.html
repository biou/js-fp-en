React

  • librairie basée composants pour les vues Web
  • syntaxe déclarative
  • philosophie fonctionnelle:
    • un composant = une fonction
    • VirtualDOM, modifications du DOM = effets de bord
    • Top-down rendering
    • Stateless functional components
    • Limiter les états
    • Architecture flux: état immuable
  • D'autres libs vont plus loin, ex: bacon.js
- component = fonction, et donc composition de fonctions pour faire une interface (syntaxe déclarative jsx)
- top-down rendering: on passe les paramètres à la fenêtre, qui va propager les informations via les props aux composants fils
- stateless functional components: nerf de la guerre dans react = la gestion des states. Très sensible aux effets de bords. Facebook encourage l'écriture de petits composants appelés stateless functional, dont la sortie dépend totalement des paramètres en entrée
- état immuable, on réaffiche lorsque l'état change. Prob de savoir si l'état a changé, avec état immuable on le sait
- bacon.js: functional reactive programming, travaille sur des flux d'événements
